# IndexedDB Adapter

The `IndexedDBAdapter` is explicitly designed for offline-first mobile web constraints (PWAs) and applications handling unusually large FeatureDrop manifests.

Rather than blocking the main thread parsing a massive `localStorage` JSON blob or waiting for a slow cellular network request, this adapter leverages the browser's asynchronous IndexedDB engine to cache and query state instantly.

## Offline Write Queues

Whenever a user interaction occurs while the device has no internet connection (e.g., they complete Checklist item #1 on an airplane), the adapter appends the event to an internal transaction queue.

Once the `online` event fires or the browser regains visibility, the adapter automatically flushes the queue via `flushQueue()` and reconciles local state with your remote backend.

## Typical Integration

```ts filename="featuredrop.config.ts"
import { IndexedDBAdapter } from 'featuredrop'

// Initialize the local-first DB adapter
export const storage = new IndexedDBAdapter({
  databaseName: 'featuredrop-client-db',
  autoSync: true,
  syncOnOnline: true,           // Automatic recovery when network returns
  syncOnVisibilityChange: true  // Automatic recovery when tab is aggressively foregrounded
})

// Optional: manually force a background sync payload
await storage.flushQueue()
```

<Callout type="info">
  **Best Practices**:
  If building an internal enterprise application for field workers (like warehouse scanners or delivery tablets), it is highly recommended to expose a manual "Force Sync" button in a debug surface that manually fires `syncFromRemote()` and `flushQueue()`, allowing users to forcefully reconcile state with headquarters if the ServiceWorker stalls.
</Callout>

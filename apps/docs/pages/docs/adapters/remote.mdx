# Remote Adapter

The `RemoteAdapter` is the standard choice for production web applications. It ensures that a user's adoption state (e.g., "Has seen the Dashboard Tour") converges perfectly across all their distinct sessions and devices.

If a user dismisses a Changelog on their mobile phone, the `RemoteAdapter` ensures the Changelog remains dismissed when they log in to their desktop browser an hour later.

## Core Capabilities

Because distributed systems are inherently unreliable, the Remote Adapter ships with enterprise-grade safeguards natively:

- **Exponential Backoff**: Failed network requests seamlessly retry with escalating delays to avoid swamping your API.
- **Circuit Breakers**: If an upstream API goes down, the adapter trips a local fast-fail circuit to prevent a runaway loop of failing network requests from hanging the browser thread.
- **Diagnostic Signals**: You can observe the adapter's `isHealthy()` signal to feed telemetry dashboards.

## Typical Integration

```ts filename="featuredrop.config.ts"
import { RemoteAdapter } from 'featuredrop'

// Initialize the remote adapter pointing to your backend endpoint
export const storage = new RemoteAdapter({
  baseUrl: '/api/featuredrop',
  retry: { 
    maxAttempts: 3, 
    baseDelayMs: 300 
  },
  circuitBreaker: { 
    failureThreshold: 5, 
    cooldownMs: 60000 // Halt requests for 1 minute if 5 consecutive errors occur
  }
})
```

<Callout type="warning">
  **Operational Recommendations**:
  Always ensure that you set request-level timeouts at the transport layer logic on your backend `/api/featuredrop` route. If the remote goes down, FeatureDrop will gracefully fall back, but you should log circuit state transitions to your monitoring provider.
</Callout>

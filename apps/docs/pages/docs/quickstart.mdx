# Quickstart

Get your first working badge in under 10 minutes. This guide covers the simplest setup using the React SDK and an in-memory storage adapter suitable for local development.

<Steps>
### Install the package

Install the base library and its React bindings via your package manager of choice.

```bash filename="Terminal"
pnpm add featuredrop
```

### Define a manifest

The manifest is your single source of truth. It defines every feature, announcement, or tour your product will surface — along with their targeting rules and expiration dates. Everything lives in your codebase, under version control.

```ts filename="manifest.ts"
import type { FeatureManifest } from 'featuredrop'

export const manifest: FeatureManifest = [
  {
    id: 'new-dashboard',
    title: 'New Dashboard',
    version: '1.0.0',
    releaseDate: '2026-02-01T00:00:00Z',
    description: 'A faster dashboard with updated real-time charts and CSV export.'
  }
]
```

### Wrap your app

Wrap your React tree with `FeatureDropProvider`. Pass it a manifest and a storage adapter. We use `MemoryAdapter` here for quick local testing — it does not persist state between page reloads.

```tsx filename="App.tsx"
import { MemoryAdapter } from 'featuredrop'
import { FeatureDropProvider, ChangelogWidget, NewBadge } from 'featuredrop/react'
import { manifest } from './manifest'

export function App() {
  return (
    <FeatureDropProvider manifest={manifest} storage={new MemoryAdapter()}>
      <div className="flex items-center gap-4">
        <span>Dashboard</span>
        {/* Renders a pulsing dot next to 'Dashboard' until the user has seen it */}
        <NewBadge featureId="new-dashboard" />
      </div>

      {/* Global slide-out changelog panel */}
      <ChangelogWidget />
    </FeatureDropProvider>
  )
}
```

<Callout type="warning">
  **Do not use `MemoryAdapter` in production.** It stores state in memory only — dismissals are lost on every page reload. Use `LocalStorageAdapter` for SPAs or `RemoteAdapter` / `PostgresAdapter` when state must persist across devices and sessions.
</Callout>

### Validate before deploying

Run these commands in your CI pipeline to catch manifest errors before they reach users:

```bash
# Validate schema and semantic rules
npx featuredrop validate --pattern "features/**/*.md"

# Quick local bundle size check
pnpm featuredrop size-check
```

</Steps>

## 1) Pick the right package entrypoint

| Import path | Use when |
| --- | --- |
| `featuredrop` | Core logic, adapters, manifest helpers, parser, analytics primitives |
| `featuredrop/react` | Provider, hooks, and built-in UI components |
| `featuredrop/schema` | Runtime validation (`validateManifest`) + Zod/JSON schema exports |
| `featuredrop/testing` | Mock manifest/storage helpers and React test wrappers |
| `featuredrop/ci` | Manifest diffing and CI summary generation |
| `featuredrop/flags` | Feature flag bridges and gating helpers |
| `featuredrop/bridges` | Slack/Discord/webhook/email/RSS notification bridge utilities |
| `featuredrop/admin` | Manifest editor + preview/admin UI blocks |
| `featuredrop/vue` / `featuredrop/svelte` / `featuredrop/solid` / `featuredrop/preact` / `featuredrop/web-components` / `featuredrop/angular` | Framework adapters |

## 2) Core runtime primitives (`featuredrop`)

```ts
import {
  isNew,
  getNewFeatures,
  getNewFeatureCount,
  getNewFeaturesSorted,
  MemoryAdapter
} from 'featuredrop'

const storage = new MemoryAdapter()
const entries = getNewFeatures(manifest, storage)
const count = getNewFeatureCount(manifest, storage)
const sorted = getNewFeaturesSorted(manifest, storage)
```

Most common helpers:

- `isNew(...)` -> single-entry eligibility gate
- `getNewFeatures(...)` -> all currently eligible entries
- `getNewFeatureCount(...)` -> count for badges/notifications
- `parseDescription(markdown)` -> markdown to sanitized HTML
- `generateRSS(manifest, options?)` -> changelog RSS XML

## 3) React SDK (`featuredrop/react`)

```tsx
import { MemoryAdapter } from 'featuredrop'
import {
  FeatureDropProvider,
  ChangelogWidget,
  NewBadge,
  useFeatureDrop
} from 'featuredrop/react'

function ReleaseActions() {
  const { dismissAll } = useFeatureDrop()
  return <button onClick={dismissAll}>Mark all read</button>
}

export function App() {
  return (
    <FeatureDropProvider manifest={manifest} storage={new MemoryAdapter()} appVersion="2.4.0">
      <NewBadge featureId="new-dashboard" />
      <ChangelogWidget />
      <ReleaseActions />
    </FeatureDropProvider>
  )
}
```

## 4) Validation and schema (`featuredrop/schema`)

```ts
import { validateManifest } from 'featuredrop/schema'

const result = validateManifest(manifest)
if (!result.valid) {
  console.error(result.errors)
  process.exit(1)
}
```

`validateManifest` checks:

- required fields (`id`, `label`, `releasedAt`, `showNewUntil`)
- duplicate IDs
- date ordering (`showNewUntil` after `releasedAt`)
- dependency cycle safety (`dependsOn`)
- URL safety (`url`, `image`, `cta.url`)
- unsafe object keys inside `meta`

## 7) CLI commands you should actually run

```bash
# Authoring + build
npx featuredrop validate --pattern "features/**/*.md"
npx featuredrop build --pattern "features/**/*.md" --out featuredrop.manifest.json

# Release safety
pnpm security-check
pnpm size-check
```

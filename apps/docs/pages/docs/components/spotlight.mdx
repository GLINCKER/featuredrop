# The Spotlight

The `<Spotlight>` component is a highly focused guidance tool. Unlike a `<Hotspot>`, which rests passively on the UI, a Spotlight aggressively commands attention by rendering a dark, semi-transparent overlay across your entire application window, leaving only the targeted element illuminated (or "spotlit") at full brightness.

## Usage & Strategy

Spotlights are the perfect middle-ground between a passive Hotspot and a fully-blocking Announcement Modal. They immediately draw the user's eye to exactly one place while still preserving the visual context of the surrounding application.

- **High-Value Actions**: Use a Spotlight when you deploy a massive new feature (like a highly-requested "Export to CSV" button) that you want 100% of your daily active users to see within 5 seconds of logging in.
- **Auto-Dismissal**: Spotlights are typically configured to dismiss the moment the user clicks the illuminated element, ensuring a frictionless transition into utilizing the new feature.

<Callout type="warning">
  **Avoid Overuse**: Because the Spotlight blocks interactions with the rest of the un-illuminated page, it is inherently disruptive. Do not use Spotlights for minor bug fixes or small copy changes.
</Callout>

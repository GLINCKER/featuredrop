# The Tour Component

FeatureDrop supports guided product onboarding through a powerful, sequenced `<Tour>` component.

Rather than flooding the user with popup announcements on login, a Tour allows you to orchestrate an interactive sequence of steps that guides the user through high-value activation flows directly in context.

## Usage & Strategy

Tours are composed of an ordered list of "Steps," which attach themselves to specific CSS selector targets visible in your application's DOM. 

- **Orchestrate Multiple Interactions**: A tour combines `Spotlight` (focused callouts) and `Spotlight Chain` (chained highlights) logic out-of-the-box.
- **Throttling Controls**: The FeatureDrop engine ensures users aren’t prompted with more than one uncompleted tour during a single session, protecting your UX.

<Callout type="warning">
  **UX Guardrails**: FeatureDrop enforces strict accessibility constraints on tours. All popovers default to `aria-modal="true"`, focus is trapped within the active step, and the user can easily exit by pressing the `Esc` key. Always give your users an obvious "Dismiss" button if they do not wish to complete the tour.
</Callout>

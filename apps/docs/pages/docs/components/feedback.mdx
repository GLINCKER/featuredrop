# The Feedback Widget

While the Feature Request board is used for long-term roadmap planning, the `<Feedback>` widget is designed for immediate, contextual sentiment capture.

It is typically mounted floating in the bottom corner of a page, or embedded directly following a new feature's completion state to ask the user "How did we do?".

## Usage & Strategy

The Feedback component is designed to be completely headless regarding *where* the data goes. You provide your own API endpoint or server action, and the component handles the complex UX of gathering the data safely.

- **Sentiment Capture**: Ships with built-in CSAT (Customer Satisfaction), NPS (Net Promoter Score), and emoji-based sentiment voting patterns.
- **Categorization**: Allows users to bucket their feedback (e.g., "Bug Report", "UI Suggestion", "Performance Issue").
- **Rate Limiting**: Built-in hooks prevent disgruntled users from spamming your API with hundreds of submissions.

<Callout type="tip">
  **Screenshot Metadata**: If your application is particularly complex, you can configure the `<Feedback>` component to optionally capture a DOM snapshot or metadata payload (like the current URL route and active Feature Flags) alongside the user's text submission to help your engineers debug faster.
</Callout>

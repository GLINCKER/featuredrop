# The Hotspot Component

A `<Hotspot>` is a subtle, localized indicator—usually rendered as a pulsing beacon or colored dot—that sits continuously over a specific UI element in your application to draw the user's eye.

## Usage & Strategy

Hotspots are the masters of **Progressive Disclosure**. Unlike an Announcement Modal that traps the user's focus immediately upon login, a Hotspot waits patiently. It simply indicates that something new exists behind a button, allowing the user to discover it naturally when they are ready to explore.

- **Lightweight Guidance**: Because they don't block the screen, you can safely deploy multiple Hotspots at once to highlight several small iterative improvements across a dense dashboard.
- **Tooltip Integration**: When hovered or clicked, a Hotspot expands into a localized Tooltip containing a title, description, and action button.
- **Composite Flows**: Hotspots are easily composable. You can configure a Changelog widget to trigger a Hotspot, or have a Checklist task direct the user to click a Hotspot.

<Callout type="info">
  **Targeting Mechanics**: Like Tours and Spotlights, Hotspots use CSS selector targeting. You simply provide the ID or class of the element you want to highlight (e.g., `target="#new-export-btn"`), and the FeatureDrop positioning engine will automatically calculate the optimal placement so it doesn't obscure the button itself.
</Callout>
